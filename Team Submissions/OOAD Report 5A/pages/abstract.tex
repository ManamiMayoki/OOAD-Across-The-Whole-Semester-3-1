This lab report presents the domain modeling for a Multi-Tenant E-Commerce System (M-TECS) using UML class diagrams. Domain modeling focuses on representing the core business concepts, entities, and relationships within the system, independent of technical implementation details. The lab exercise involved identifying key domain classes—such as Vendor, Customer, Product, Order, Discount, Themes, and Navigation Menu—based on previously developed use case diagrams. Each class was analyzed to determine its essential attributes and relationships with other classes, with careful attention to multiplicities and business constraints. The domain model was constructed using Lucidchart, a web-based diagramming tool, to create a visual representation that bridges the gap between high-level system requirements and detailed design. The resulting domain model serves as a foundation for subsequent design phases, ensuring that all stakeholders share a common understanding of the business domain before implementation begins.

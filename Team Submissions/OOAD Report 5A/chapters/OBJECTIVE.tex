\chapter{OBJECTIVE}\label{obj}

The primary objectives of this lab exercise are as follows:

\begin{enumerate}
    \item\textbf{Identify Core Domain Classes:} Extract and identify the primary business entities from the Multi-Tenant E-Commerce System based on previously developed use case diagrams and system requirements.
    
    \item\textbf{Define Essential Attributes:} Determine the business-relevant attributes for each domain class, focusing on properties that describe the entity's state and identity within the system.
    
    \item\textbf{Establish Relationships and Multiplicities:} Identify and model the associations between domain classes, including proper multiplicities (e.g., one-to-many, many-to-many) that reflect real-world business rules.
    
    \item\textbf{Document Business Constraints:} Capture and represent domain-specific business rules and constraints (e.g., "A customer can only review products they have purchased") within the class diagram.
    
    \item\textbf{Demonstrate Domain Modeling Proficiency:} Apply object-oriented analysis principles to create a conceptual model that is independent of implementation concerns such as databases, user interfaces, or technical frameworks.
    
    \item\textbf{Visualize Using Industry-Standard Tools:} Utilize Lucidchart to create professional UML class diagrams that clearly communicate the domain structure to both technical and non-technical stakeholders.
    
    \item\textbf{Establish Foundation for Design:} Create a domain model that will serve as the blueprint for subsequent design activities, including database schema design and detailed class diagrams with methods and visibility.
\end{enumerate}
